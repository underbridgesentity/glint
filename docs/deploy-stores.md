# Shipping Glint to the App Store & Google Play

The Customer (`apps/customer`) and Technician (`apps/technician`) apps are Expo
apps, built and submitted with **EAS**. The config is in place; this is the
runbook.

## What you need to provide (one-time)
- **Expo account** — free, at [expo.dev](https://expo.dev).
- **Apple Developer Program** — $99/yr ([developer.apple.com](https://developer.apple.com)).
- **Google Play Console** — $25 once ([play.google.com/console](https://play.google.com/console)).

## 1. Link the apps to EAS
```bash
npm i -g eas-cli
eas login

# from each app dir:
cd apps/customer && eas init     # creates the EAS project, fills extra.eas.projectId
cd ../technician && eas init
```
`eas init` writes the real `projectId` into each `app.json` (replacing
`REPLACE_WITH_EAS_PROJECT_ID`) and sets `owner` to your Expo username.

## 2. First builds (get it on your phone)
```bash
# internal builds you can install directly
cd apps/customer
eas build --profile preview --platform ios        # TestFlight / simulator
eas build --profile preview --platform android     # installable APK
```
Repeat in `apps/technician`. The `preview` profile (see `eas.json`) gives an
Android APK and an iOS internal build — fastest way to hold it in your hand.

## 3. Store (production) builds
```bash
eas build --profile production --platform all
```
EAS manages signing credentials for you (Apple + Android keystore) on first run.

## 4. Submit
```bash
eas submit --profile production --platform ios       # to App Store Connect
eas submit --profile production --platform android    # to Play Console
```

## Store listing checklist (per app)
- **Privacy policy URL:** https://glint-blond.vercel.app/privacy ✅ (live)
- **Account deletion:** in-app under Profile → Delete account ✅ (Apple requires this)
- Screenshots (use the real screens; iPhone 6.7" + Android phone sizes)
- App description, keywords, category (Lifestyle / Auto)
- Data-safety form (Play) / privacy "nutrition label" (Apple): we collect name,
  phone, email, vehicle info, photos; payments via PayFast; no data sold.
- Support email + marketing URL (https://glint-blond.vercel.app)

## Assets in place
- `assets/icon.png` (1024²), `adaptive-icon.png`, `splash.png`, `favicon.png` —
  Carbon + Electric-Lemon mark, generated by `scripts/gen-icons.mjs`.
- Bundle IDs: `co.za.glint.customer`, `co.za.glint.technician`.
- Camera usage strings + permissions declared (for proof photos).

## Notes
- **Phone OTP** needs an SMS provider (Twilio/MessageBird) enabled in Supabase →
  Auth before submission, or reviewers can't sign in. Provide a demo account or
  enable email sign-in for review.
- Monorepo: EAS builds from the repo with the root `.npmrc` (`node-linker=hoisted`),
  which is required for Expo + pnpm workspaces.
- OTA updates: `eas update --channel production` (channels are set in `eas.json`).
